the objective of this project is to build a straight forward POS Application that has primary support for tablet devices, but is cross-platform and can be used in mobile and desktop hardware.

I've made an outline of the improvements that will be added along the way for each versions:

V0 - Simple add-to-cart and checkout flow with controls to add items that the user wants, set prices for each items, and add categorization. there will also be a way to add a fix charge to the cart for tax, service charge, or any financial purposes. I will be allowing the user to pay with cash or card. Cash payments will be recorded without any reference ID, card payments on the other hand will require a reference ID. reference id will be optional. when a cart is checked-out, the system will automatically record the transaction amount, items ordered, extra fee, and transaction type. these recorded transaction can be exported based on the selected day.

V1 - Users will be able to add bundled orders from existing items and set prices for them. they will also be able to do add-ons to the order if they wish to by creating a simple linking process between add-ons and items. On top of that, a dashboard is also going to be presented with projected sales chart, daily/weekly/monthly comparison. Users will also be able to apply discounts or any type of addition/deduction to the transaction before checkout. finally, i will be introducing Out-Of-Stock so users don't have to remove the item from their item list if it's not available

V2 -  We will be introducing user logins that will allow owners to track who sold what. We will also be introducing additional reporting tools that the user can use. We will also attempt to build the online sync so owners have a central dashboard that have all of the analytics at a glance.

This application is meant to be straight forward and remove all of the complications of a POS system. we will be tailoring to one shop and will be reviewing suggestions and issues along the way to improve the application.

Application will only be released when the v1 is established. at the very least, the v1 will serve as a viable, usable application for the test subject.
